# 📖 About I EXPENSE

I EXPENSE is a **calendar-first expense management application** that reimagines how personal finance should feel.

Instead of displaying endless lists of transactions, I EXPENSE transforms every day into a visual financial snapshot. Users can instantly understand how much they spent, how much they earned, and how their month is progressing—all directly from the calendar.

The application was built with one simple philosophy:

> **Money is remembered by days, not by timestamps.**

When we think back, we rarely remember:

> "I spent ₹250 at 3:42 PM."

Instead, we remember:

> "What happened on the 18th? Why did I spend so much that day?"

That simple thought became the foundation of this project.

Built as a **Progressive Web App (PWA)**, I EXPENSE works across Android, iPhone, Windows, macOS, and Linux without requiring installation from an app store.

No sign-up.

No subscription.

No ads.

Just open the website and start tracking.

# 💜 Developer's Touch

Hello! 👋

I'm **Akash**.

And to be completely honest...

**I'm absolutely terrible at tracking my expenses.** 😅

Every month I'd confidently tell myself,

> "This month I'll finally manage my money properly."

A few food deliveries later...

A random Amazon purchase...

One gadget I definitely didn't need...

...and suddenly I'd be staring at my bank balance wondering where everything disappeared.

Instead of downloading another expense tracker...

**I decided to build one that I'd actually enjoy using.**

---

## 📅 Why a Calendar?

Most finance apps organize everything into long lists of transactions.

Personally...

that's never how I remember my spending.

I remember **days**.

> "That one GPay trasaction on thursday to an unknown seller"
 
> "What happened on Sunday?"

> "Why was the 18th so expensive?"

> "Didn't I buy something yesterday?"

So instead of making transactions the center of the application...

I made the **calendar** the main character.

Every single day becomes its own financial story.

---

## 🌐 Why a Website?

Here's the funny part...

I'm primarily an **Android Developer**.

But...

I use an **iPhone.**

Yeah...

The irony isn't lost on me. 😂

I could've built this as a native Swift application.

But then I'd have to deal with Apple Developer licensing, provisioning profiles, code signing, TestFlight...

...and honestly, I just wanted to track my expenses.

So I thought,

> **Why not build something everyone can use?**

Whether you're using

- Android
- iPhone
- Windows
- macOS
- Linux

...just open the website and start using it.

No installation.

No accounts.

No subscriptions.

Just budgeting.

---

## 📄 Why Everything Lives Inside One File?

Most people expect projects to have dozens of folders and hundreds of files.

This project intentionally doesn't.

Everything lives inside **one `index.html`**.

Not because I didn't know how to split it.

Because I wanted to challenge myself.

I wanted to answer one question:

> **"How much can I build using only HTML, CSS and Vanilla JavaScript?"**

No React.

No Vue.

No Angular.

No npm.

No frameworks.

No libraries.

Just pure web technologies.

The result is a complete finance application with calendar management, analytics, search, goals, budgets, smart insights, offline support, animations and even a game—all built without external dependencies.
